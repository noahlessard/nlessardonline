The train cam project continues to advance (like a train along rails perhaps?) mostly with a lot of performance improvements. The Raspberry Pi 3 cpu didn't seem to have the guts to run the x11 window compositor managing both a transparent foot terminal rendering TUI and the vlc train video. So, experiments are ongoing using mpv, which allows for a IPC method to just change the video frame in memory before it gets rendered. So far, they have been successful and much faster...
